## Contexte

ORACLE v2 est un système de trading neuromorphique multi-strates (12 strates, parliament Hebbian, brainstem circuit breaker, safety kernel). L'analyse initiale lui attribue un score de 5.5/10 en raison de bugs techniques, d'une absence de persistence, et d'un manque de backtesting réel. ## Phase 1 — Fondations (bugs critiques)

### 1.1 MACD Signal line — `brain/feature_layer.py`

**Problème :** Le calcul du MACD signal line est incorrect. La list comprehension ne calcule pas correctement les EMA différentielles — elle recalcule l'EMA depuis le début de la série à chaque itération au lieu d'utiliser les valeurs MACD incrémentales.

**Fix :** Calculer d'abord la série MACD complète (ema_fast - ema_slow pour chaque point), puis appliquer EMA(9) sur cette série.

```python
# Avant (incorrect)
macd_signal = self.compute_ema([ema_fast - self.compute_ema(closes[:i+1], 26)
                                for i in range(26, len(closes))], 9)

# Après (correct)
macd_series = []
for i in range(26, len(closes)):
    ef = self.compute_ema(closes[:i+1], 12)
    es = self.compute_ema(closes[:i+1], 26)
    macd_series.append(ef - es)
macd_signal = self.compute_ema(macd_series, 9) if len(macd_series) >= 9 else 0.0
```

### 1.2 `import time` top-level — `brain/feature_layer.py`

**Problème :** `import time` est à l'intérieur de la méthode `extract()` — re-importé à chaque appel.

**Fix :** Déplacer au top-level du module.

### 1.3 asyncio deprecated — `brain/narrator.py`

**Problème :** `asyncio.get_event_loop().run_in_executor(None, lambda: ...)` est déprécié depuis Python 3.10 et lève des warnings.

**Fix :**
```python
# Avant
msg = await asyncio.get_event_loop().run_in_executor(
    None, lambda: self._llm_client.messages.create(...)
)

# Après
msg = await asyncio.to_thread(
    self._llm_client.messages.create, ...
)
```

### 1.4 Prompt caching Anthropic — `brain/narrator.py`

**Problème :** Le `SYSTEM_PROMPT` (≈200 tokens) est re-envoyé à chaque appel Anthropic sans caching. Sur 100 appels/session, c'est 20 000 tokens inutiles.

**Fix :** Utiliser `cache_control={"type": "ephemeral"}` sur le system prompt (TTL 5 min Anthropic) :

```python
messages_create(
    model="claude-haiku-4-5-20251001",
    max_tokens=120,
    system=[{
        "type": "text",
        "text": self.SYSTEM_PROMPT,
        "cache_control": {"type": "ephemeral"}
    }],
    messages=[{"role": "user", "content": full_ctx}]
)
```

### 1.5 Working Memory TTL — `brain/working_memory.py`

**Problème :** Le buffer accepte des signaux sans expiry — un signal de 10 minutes peut former un faux consensus avec un signal récent.

**Fix :** Ajouter `max_signal_age_seconds: int = 90` dans le constructeur. Dans `get_consensus()`, filtrer les signaux périmés avant de compter le consensus.

```python
def get_consensus(self) -> Optional[tuple[str, float]]:
    now = time.time()
    valid = [s for s in self.buffer 
             if now - s.timestamp <= self.max_signal_age_seconds]
    if len(valid) < self.required_consensus:
        return None
    # ... reste inchangé
```

**Config :** Ajouter `WORKING_MEMORY_TTL_SECONDS: int = 90` dans `OracleConfig`.

### 2.2 Hebbian weights persistés — `brain/parliament.py`

**Modification :** `HebbianWeightManager.__init__()` accepte un `HebbianRepository` optionnel. Au démarrage, `load()` charge les poids depuis SQLite. Après chaque `update()`, `save_weight()` persiste immédiatement.

```python
class HebbianWeightManager:
    def __init__(self, strate_names: list, repo: HebbianRepository = None):
        self.repo = repo
        self.weights = {s: 1.0 for s in strate_names}
        if repo:
            saved = repo.load_weights(strate_names)
            self.weights.update(saved)

    def update(self, strate: str, was_profitable: bool, pnl_pct: float) -> None:
        # ... logique existante inchangée ...
        if self.repo:
            self.repo.save_weight(strate, self.weights[strate])
```

### 2.3 ATR-based SL/TP — `brain/safety_kernel.py`

**Problème :** SL fixe entre 0.5% et 2% — ignore la volatilité du marché. En marché très volatile (ATR élevé), le SL est trop serré. En marché calme, il est trop large.

**Fix :** `SafetyKernel.validate()` accepte un `atr_pct: float` optionnel :

```python
def compute_atr_sl(self, atr_pct: float) -> float:
    """SL = 1.5× ATR, clampé entre min et max config."""
    sl = atr_pct * 1.5
    return max(self.min_sl_pct, min(self.max_sl_pct, sl))
```

L'ATR est fourni par la FeatureLayer existante — pas de nouvelle dépendance.

## Phase 3 — Intelligence & Refactoring

### 3.1 Backtest vectorisé — `backtest/`

**`backtest/data_loader.py`** :
- `fetch_historical(symbol, timeframe, days=90)` → appelle Binance REST (ccxt)
- Cache local JSON dans `oracle_v2/backtest/cache/` pour éviter re-fetch

**`backtest/engine.py`** :
```
BacktestEngine(config, strates, parliament)
  .run(symbol, timeframe, days=90) → BacktestResult

BacktestResult:
  - trades: list[dict]  (features + direction + pnl_pct)
  - metrics: {winrate, sharpe_ratio, max_drawdown, total_trades}
  - strate_scores: dict  (winrate par strate)
```

Logique : replay bougie par bougie, SL/TP ATR-based, pas de lookahead bias.

### 3.2 XGBoost — `brain/predictive_layer.py`

**Training flow :**
1. `BacktestEngine.run()` → génère un DataFrame `features + outcome`
2. `PredictiveLayer.train(df)` → XGBoost avec early stopping (80/20 split)
3. Modèle sauvé dans `oracle_v2/models/xgb_oracle.json`
4. `PredictiveLayer.predict()` → XGBoost réel si modèle présent, heuristique sinon

**Features utilisées :** rsi, macd, macd_signal, bb_position, volume_ratio, atr, orderbook_imbalance, trend_score, top_3_hebbian_weights.

**Commande d'entraînement :**
```bash
python -m oracle_v2.backtest.train --symbol BTCUSDT --days 90
```
